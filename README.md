# PulseBoard — AI-Powered Observability Platform

PulseBoard is a state-of-the-art microservices observability platform that combines real-time monitoring with AI-driven incident analysis. It doesn't just tell you that your service is down; it tells you **why** by correlating metrics, logs, and recent GitHub commits using Gemini AI.

![PulseBoard Dashboard Mockup](https://raw.githubusercontent.com/lucide-icons/lucide/main/icons/activity.svg)

---

## 🚀 Features

- **Real-time Monitoring**: Sub-second latency tracking via Poller microservice.
- **AI Root Cause Analysis**: Automated incident reports generated by Gemini 2.5 Flash, identifying probable causes from logs and code changes.
- **Smart Log Ingestion**: High-throughput log ingestion with asynchronous vector embeddings for semantic search.
- **GitHub Correlation**: Automatically fetches recent commits during incidents to identify regression-inducing changes.
- **Multi-Channel Alerting**: Native support for Discord, Slack, and generic webhooks.
- **Live Dashboard**: High-performance Next.js 14 frontend with SSE (Server-Sent Events) for real-time telemetry.

---

## 🏗️ Architecture

PulseBoard is built as a **NestJS Monorepo** with four specialized microservices communicating via **RabbitMQ** and **Redis**:

- **API Gateway (3001)**: Central entry point for Auth, CRUD, and SSE streams.
- **Poller Service (3002)**: Distributed health checker with dynamic cron scheduling.
- **Ingestor Service (3003)**: High-speed telemetry collector for MongoDB and vector storage.
- **Alert Service (3004)**: AI brain that handles incident reports and third-party dispatch.

---

## 🛠️ Tech Stack

- **Backend**: NestJS 11, TypeScript 5, TypeORM (PostgreSQL), Mongoose (MongoDB).
- **Frontend**: Next.js 14 (App Router), Tailwind CSS, shadcn/ui, Recharts.
- **Infra**: RabbitMQ (Message Broker), Redis (Caching/PubSub), Docker.
- **AI**: Vercel AI SDK, Google Gemini (Primary), Groq (Fallback).

## 🚦 Getting Started

### 1. Prerequisites
- Node.js 20+
- Docker & Docker Compose
- Gemini API Key (from [AI Studio](https://aistudio.google.com/))

### 2. Infrastructure Setup
Spin up the local databases and message broker:
```bash
docker compose up -d
```

### 3. Environment Configuration
Copy the `.env.example` (or use the values from AGENTS.md) into a `.env` file in the root.

### 4. Installation
```bash
npm run install:all
```

### 5. Database Initialization
```bash
# Run migrations
npm run db:migrate

# Seed initial data (optional)
npm run seed
```

### 6. Start Development
```bash
npm run dev
```
- Frontend: `http://localhost:3000`
- API Gateway: `http://localhost:3001`
